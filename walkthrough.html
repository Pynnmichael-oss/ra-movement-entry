User Guide

How to enter, review, and export petroleum movement data into RightAngle (ETRM).

Overview

This tool has two ways to get movements into the system. Use whichever fits the document you have:

1
Manual Entry (index.html)

Type movement data directly into a form. Best for one-off entries or when you already know the field values.

2
PDF Processor (email-processor.html)

Upload a petroleum statement or delivery ticket PDF. Claude AI reads the document and pre-fills the fields for you to review before saving.

3
Export to RightAngle

When your movements look correct, click Export Pending (.xlsx) on the Manual Entry page. Download the file and import it into RightAngle.

Data Storage

All movements are saved to a SharePoint list named RA_Movements. Data persists across sessions and is visible to all users on the site — no local storage is used.

Manual Entry

Open index.html from the SharePoint document library. On first load you will be asked for your name — this is recorded on each movement you create.

1
Fill in the required fields

Fields marked with an asterisk (*) must be completed. At minimum: ExternalDocumentNumber, MovementDate, Location, MovedProduct, NetVolume, and VolumeUOM.

2
Fill in optional fields as needed

Type, deal numbers, counterparty names, gravity values, and notes can all be added below the divider. Leave anything blank that does not apply.

3
Click + Add Row

The movement is saved to SharePoint and appears in the table below. The status will be PENDING.

4
Correct mistakes using Edit

Pending rows have an Edit button. Click it to open the edit modal, make changes, and save. Once a row is exported it becomes read-only.

5
Remove a row using Del

Click Del on any pending row to permanently delete it from SharePoint. This cannot be undone.

Important — MovementDate Format

Enter the date and time of the physical movement, not today's date. RightAngle uses this to post the movement to the correct accounting period. If you only know the date, set the time to 12:00 AM.

Field Reference

Every field the form accepts and what it maps to in RightAngle:

Field Required Description
ExternalDocumentNumber Required Unique ID for this movement (e.g. invoice number, ticket number). Used to prevent duplicate imports in RightAngle.
MovementDate Required Date and time the physical movement occurred. Format: date + time picker.
Location Required RightAngle location code or facility name where the movement took place (e.g. TANK-101).
MovedProduct Required Product name as it appears in RightAngle (e.g. Crude Oil WTI, Gasoline, Diesel).
NetVolume Required Net volume of the movement as a plain number. No commas or units.
VolumeUOM Required Unit of measure: bbl (barrels), gal (gallons), or mcf (thousand cubic feet).
Type Optional Movement type: Production, Receipt, Delivery, or Inventory.
GrossVolume Optional Gross volume before temperature/gravity correction.
SpecificGravity / APIGravity Optional Product quality measurements. Leave blank if not on the source document.
Status Optional RightAngle matching status (separate from the app's Pending/Sent status). Often left blank.
InternalDocumentNumber Optional RightAngle internal reference number if known.
MovementDocumentDate Optional Date printed on the source document (may differ from MovementDate).
ReceiptDealNumber / ReceiptDealDetail / ReceiptBA Optional RightAngle deal number, detail line, and business associate for the receipt leg.
DeliveryDealNumber / DeliveryDealDetail / DeliveryBA Optional RightAngle deal number, detail line, and business associate for the delivery leg.
TaxOrigin / TaxDestination Optional Tax jurisdiction codes for the movement origin and destination.
CarrierBusinessAssociate Optional Carrier or pipeline operator moving the product.
LiftingNumber / ReasonCode Optional Supplementary reference fields used by some RightAngle configurations.
MovementExpenses Optional Any associated expenses. Defaults to 0 in the export if left blank.
Notes Optional Free-text notes. Not imported into RightAngle — visible in this tool only.
PDF Processor

Open email-processor.html. This page uses the Claude AI API to read a PDF and extract movement fields automatically.

1
Confirm your API key is saved

The Anthropic API key is pre-filled at the top of the page. Click Save Key once to store it. You only need to do this the first time you use the page on a new browser.

2
Upload your PDF

Drag the file onto the drop zone or click it to browse. Supported files: petroleum statements, delivery tickets, invoices — any PDF with movement data.

3
Click Parse with Claude

The PDF is sent to Claude AI for reading. This typically takes 5–15 seconds depending on PDF size. A spinner shows while it is processing.

4
Review and correct the extracted fields

A form appears with the values Claude found. Check each field — AI extraction is accurate but not perfect. Correct anything that looks wrong before saving. Fields marked FIXED (VolumeUOM and Type) are hardcoded and cannot be changed here.

5
Click Save to Movements

The movement is posted to SharePoint and a confirmation message appears. Click View All Movements to return to the main table.

Tip — What Claude Extracts

Claude looks for: invoice/document number, document date, delivery location, product name, billed volume, and the counterparty or supplier name. Fields that cannot be found on the PDF are left blank for you to fill in manually.

Note — PDF Processor Defaults

Movements created via the PDF Processor are always saved with VolumeUOM = bbl and Type = Delivery. If the movement has a different unit or type, edit the row on the Manual Entry page after saving.

Exporting to RightAngle

Exporting is done from the Manual Entry page (index.html). The export button is only active when at least one pending movement exists.

1
Review all pending rows

Check that every PENDING row is correct. Once exported, rows become read-only and cannot be edited or deleted.

2
Click Export Pending (.xlsx)

A file named RA_movements_YYYY-MM-DD.xlsx downloads immediately. All pending rows are included. The app then marks them as SENT in SharePoint.

3
Import the file into RightAngle

In RightAngle, go to the movement import screen and upload the downloaded .xlsx file. The file contains XML data in the format RightAngle expects — do not open or re-save it in Excel first, as this corrupts the XML content.

Do Not Re-Save in Excel

The .xlsx file contains raw XML data, not a normal spreadsheet. Opening it in Excel and saving it again will destroy the XML formatting and the import will fail. Import the file into RightAngle directly from your Downloads folder.

Row Statuses

Every movement row has one of two statuses:

P
PENDING

The movement has been saved but not yet exported. It will be included in the next export. You can edit or delete pending rows.

S
SENT

The movement was included in a completed export. The row is dimmed and read-only. The export timestamp is shown below the badge. Sent rows are kept permanently for your records.

FAQ

I added a row but the table shows "Connecting to SharePoint..."

The page is still loading existing movements from SharePoint on startup. Wait a moment and the table will populate. If the error banner appears, check that you are logged into your SharePoint account and that the RA_Movements list exists on the site.

I get a "Not authenticated" error

The app must be opened from within SharePoint — navigate to the document library and click the file, rather than opening it from a local folder. SharePoint session cookies are required for the REST API to work.

Claude couldn't extract a field from my PDF

Some PDFs use scanned images rather than selectable text, which makes extraction harder. If a field is blank after parsing, type the value in manually before saving. You can also click Re-parse to try again with the same file.

I exported by mistake — can I re-export?

Once rows are marked SENT they cannot be re-exported from this tool. To re-import into RightAngle, use the previously downloaded file. If you have lost the file, contact your RightAngle administrator to delete the already-imported movements so they can be re-submitted.

The ExternalDocumentNumber already exists in RightAngle

RightAngle uses ExternalDocumentNumber to detect duplicates. If an import fails with a "duplicate document" error, the movement was already imported. Check whether the export was already processed before re-submitting.

I need to change a movement after it was exported

Sent rows are locked in this tool. To correct a movement that has already been imported into RightAngle, you will need to edit or reverse it directly in RightAngle. Ask your ETRM administrator if you are unsure how to do this.
